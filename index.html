What it does

DevTools shows you styles. StyleGrab hands them to you.

Authored CSS, not computed soup

Reads the actual rules the developer wrote — merged by real cascade and specificity — with a rendered-styles fallback when stylesheets are locked.

:hover, @media, @keyframes included

One copy grabs pseudo-classes, pseudo-elements, responsive breakpoints and the animations an element references.

Four output dialects

Clean CSS, Tailwind classes (with hover: and md: prefixes), React/JSX style objects, or styled-components.

Portable by default

var(--tokens) get resolved to real values and inherited typography is folded in, so pasted snippets look right anywhere.

Pin, edit, measure

Pin the panel and edit CSS with live preview on the page. Alt-hover measures pixel distances between elements.

Palette & asset extraction

Pull a page's full color palette, images and inline SVGs in one click. Export any element straight to CodePen.

FAQ

Questions people actually ask.

How is this different from DevTools?

DevTools shows one rule at a time and makes you reassemble the cascade by hand. StyleGrab merges everything that applies to an element — including :hover states, breakpoints and keyframes — into one paste-ready block, then converts it to your framework's dialect.

Is the free tier actually free?

Yes, permanently: hover-inspect, full CSS copy, box-model overlay and the eyedropper never expire and never nag. Pro adds the framework exports, component copy, live editing and extraction tools.

What data do you collect?

None. The extension runs entirely in your browser, makes zero network requests during use, and only touches pages where you explicitly invoke it (activeTab permission). Checkout, if you buy, happens on a Stripe-backed page — we never see your card.

Does it work on React / Vue / Tailwind sites?

Yes — it reads the browser's stylesheet model, so the source framework doesn't matter. It also understands modern CSS plumbing: @layer, adopted stylesheets and shadow-root styles.

Copying styles — is that okay?

StyleGrab is a learning and productivity tool, the same way View Source and DevTools are. Use it to study techniques, debug, and move your own work between projects. Respect other people's IP; don't clone whole products.
